# The HOA / ТСЖ ("Цифровой Подъезд")

## Essence
Информирование, утилитарность, низкая тех. грамотность.

## Users
- Председатель (Admin)
- Жильцы (User)
- Арендаторы (Guest)

## Main pain
- "Когда дадут воду?"
- "Кому звонить, если прорвало?"
- "Как оплатить?"

## UX expectations
- Крупный шрифт
- Минимум кнопок
- Работает с телефона бабушки

## Configuration profile
| DB Param | Value | Why |
| --- | --- | --- |
| audience_groups | ["building_1", "building_2"] | Разные новости для разных подъездов. |
| dashboard_template | "home-hoa" | Закрепленные блоки: Important + Directory. |
| dashboard_edit | admin: manage, residents: view | Жильцы не меняют раскладку. |
| user.ttl | null / 1 year | Жильцы меняются редко, арендаторы чаще. |
| auth_method | Email Link / QR | Пароли забывают, вход по "магической ссылке". |
| directory_focus | Оплата + Камеры + Чаты | Быстро найти сервисы дома. |

## Test case
"Пенсионерка должна зайти, увидеть КРАСНОЕ объявление об отключении воды и нажать кнопку вызова сантехника. Она не должна иметь возможность скрыть это объявление."
